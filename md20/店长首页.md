店长首页 H5（Manager Dashboard）

Wireframe · Text Version

一、整体结构（单屏优先）
┌──────────────────────────────┐
│ 顶部信息区                   │
├──────────────────────────────┤
│ 今日核心待办（强制区）       │
├──────────────────────────────┤
│ 今日执行 & 异常（辅助区）    │
├──────────────────────────────┤
│ 本周排班状态（只读）         │
└──────────────────────────────┘
│ 底部 Tab 导航                │
└──────────────────────────────┘

二、顶部信息区（Header）
👋 早安，店长
📅 2025-12-20（周五）


固定高度

不滚动

不放按钮

不放统计

作用：给“时间感”，不是给功能

三、今日核心待办（⭐ 第一优先级）

这个区块必须始终在首屏，
两张卡片：巡店 + 现金

1️⃣ 巡店卡（Inspection Card）
┌──────────────────────────────┐
│ 🔍 今日巡店                  │
│                              │
│ 已完成： 1 / 2               │
│ 状态： 🟡 还需 1 次巡店       │
│                              │
│ [ 去巡店 ]                   │
└──────────────────────────────┘

状态规则（仅文字 + 颜色）

0 / 2 → 🔴 未巡店

1 / 2 → 🟡 未达标

≥2 / 2 → 🟢 已完成

点击行为

未完成 → 跳转【巡店填写页】

已完成 → 跳转【今日巡店记录】

2️⃣ 现金卡（Cash Card）
┌──────────────────────────────┐
│ 💸 今日现金                  │
│                              │
│ 状态： 🟡 未日结              │
│ 理论现金： 12,500,000 VND    │
│                              │
│ [ 去日结 ]                   │
└──────────────────────────────┘

状态文本（固定几种）

🔴 未开始

🟡 进行中

🟠 有差额待说明

🟢 已完成

点击行为

未完成 → 跳转【现金日结填写页】

已完成 → 跳转【今日现金明细】

六、底部 Tab 导航（固定）
┌──────────────────────────────┐
│ 🏠 首页 │ 🔍 巡店 │ 💸 现金 │ 👤 我的 │
└──────────────────────────────┘

行为约束

首页：只展示 Dashboard

巡店：直接进入【巡店填写 / 今日巡店】

现金：进入【现金流水 + 日结】

我的：个人信息 / 退出

七、页面交互约束（非常重要）
❌ 不允许

首页出现列表滚动

首页出现配置按钮

首页出现历史日期选择

首页出现“管理设置”

✅ 强制

首页所有按钮 ≤1 次点击进入目标页

状态颜色 + 文案必须同步

首屏必须看完“巡店 + 现金”

八、验收标准（非常实在）

打开首页 3 秒内，
店长必须能回答这两个问题：

今天巡店做完了吗？

今天现金交了吗？

如果能 → 这个首页是合格的。

九、一句总结

这是一个“逼管理动作发生”的首页，
而不是“展示数据”的首页。
