H5 巡店页（Inspection – Manager）

Wireframe · Text Version

一、页面定位（非常重要）

页面类型：即时操作页

使用时机：巡店当下

使用频率：每天 ≥2 次

页面原则：一次只做一件事

二、整体结构（单屏优先）
┌──────────────────────────────┐
│ 顶部：页面标题               │
├──────────────────────────────┤
│ 巡店状态提示                 │
├──────────────────────────────┤
│ 巡店拍照（核心）             │
├──────────────────────────────┤
│ 巡店说明（可选）             │
├──────────────────────────────┤
│ 发现问题？（快捷）           │
├──────────────────────────────┤
│ 提交按钮（固定底部）         │
└──────────────────────────────┘

三、顶部区域（Header）
← 返回      🔍 巡店记录


只允许：

返回

标题

❌ 不允许功能按钮

❌ 不允许日期选择

四、巡店状态提示（只读）
📅 今日巡店
已完成： 0 / 2 次


文案目的：
提醒“你现在在做第几次巡店”

不可点击

不可修改

五、巡店拍照区（⭐ 核心，必须完成）
模块标题
📷 巡店现场照片（必填）

UI 结构
[ 📸 拍照 ]


拍照后：

[ 照片 1 ] [ 照片 2 ] [ + ]

强约束规则（必须实现）

✅ 只能调用相机

❌ 禁止相册上传

✅ 至少 1 张

建议上限：5 张

没有照片 = 不存在巡店

六、巡店说明（可选，弱约束）
✏️ 巡店说明（可选）
[ 输入框 ]

占位提示文案（非常重要）

如：吧台正常 / 后厨需注意卫生 / 高峰前检查

规则：

可空

不限制长度

不参与是否完成判断

七、发现问题？（快捷入口，非强制）

这是把巡店 → 问题管理连起来的关键

UI 结构
⚠ 是否发现问题？
[ ☐ 无问题 ]   [ ➕ 记录问题 ]

若选择「➕ 记录问题」

弹出层 / 内嵌表单（极简）

问题类型：
( ) 异常
( ) 货损
( ) 错做

问题说明：
[ 输入框 ]

[ 使用巡店照片 ] 或 [ 重新拍照 ]


规则：

问题可独立保存

自动关联本次巡店

不影响巡店提交

八、提交按钮（固定在底部）
[ ✅ 提交巡店 ]

按钮状态规则

❌ 未拍照 → 禁用

❌ 正在上传 → loading

✅ 条件满足 → 可点击

九、提交后的系统行为（不可省略）

提交成功后：

弹出成功提示：

✔ 巡店已提交，等待老板审核


自动返回：

店长首页
或

今日巡店记录页

本次巡店状态：

submitted

不计入次数，直到审核通过

十、禁止行为（一定要限制）

❌ 修改巡店时间

❌ 修改已提交巡店

❌ 上传历史照片

❌ 补填昨天巡店

十一、验收标准（非常实在）

让一个没培训的店长，用 30 秒完成一次巡店：

打开页面

拍照

提交

如果能 → 页面合格。

十二、一句话总结

这是一个“现场动作页”，不是“填写表单页”
