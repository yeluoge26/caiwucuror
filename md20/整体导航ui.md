整体导航设计原则（先定死）
1️⃣ H5 ≠ PC

H5 = 行为入口

PC = 功能目录

2️⃣ 导航必须服务角色

店长：执行 + 管理

老板 / 财务：查看 + 审核

3️⃣ 不再允许「一个导航承载所有东西」
二、H5 导航结构图（店长端）

使用频率：每天高频
使用姿势：单手

✅ H5 导航结构（最终）
┌───────────────────────────┐
│        页面内容区          │
└───────────────────────────┘
│ 🏠 首页 │ 🔍 巡店 │ 💸 现金 │ 👤 我的 │
└───────────────────────────┘

🔹 底部 Tab（只允许 4 个）
1️⃣ 🏠 首页（Manager Dashboard）

默认入口 / 打开即见

内容：

今日巡店（0/2，红黄绿）

今日现金（日结状态）

今日任务

今日问题

本周排班状态（只读）

👉 这是店长真正的“控制台”

2️⃣ 🔍 巡店

直接干活，不用找

子页面（内部跳转，不是导航）：

新建巡店（拍照）

今日巡店记录

❌ 不放历史列表
❌ 不放筛选

3️⃣ 💸 现金

只放“今天的钱”

子页面：

今日现金流水（cash）

现金日结填写页

❌ 不放报表
❌ 不放历史月份

4️⃣ 👤 我的

个人相关

内容：

我的排班（只看）

退出登录

📌 尽量保持“冷区”

❌ H5 明确禁止出现的内容

分类管理

支付方式管理

用户管理

系统设置

报表大图

👉 一旦出现在 H5，一定会降低使用率
